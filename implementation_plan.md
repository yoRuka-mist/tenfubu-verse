# 実装計画書 - 新カード「amandava」追加と「退職代行」修正

## 1. 目的
新フォロワー「amandava」を追加し、相手フォロワーのHPを操作する新しい戦術を導入する。また「退職代行」の画像を正しいものに更新する。

## 2. 実装内容
### 2.1. カード定義の更新 (`engine.ts`)
- **退職代行 (`s_resignation_proxy`)**:
    - `imageUrl` を `/cards/taisyokudaiko.png` に変更。
- **amandava (`c_amandava`)**:
    - **ID**: `c_amandava`
    - **コスト**: 5
    - **スタッツ**: 2/3
    - **効果**:
        - **ファンファーレ**: ランダムな相手フォロワー2体のHPを1にする。
        - **進化時**: 相手のフォロワーすべてに1ダメージ。
    - **画像**: `/cards/amandava.png`

### 2.2. エンジン拡張 (`types.ts`, `engine.ts`)
- **新規効果 `RANDOM_SET_HP`**:
    - `value` (対象数) と `value2` (変更後のHP) を指定可能にする。
    - 相手のボードから `value` 体をランダムに選び、`currentHealth` を `value2` に書き換える。

### 2.3. ビジュアル演出 (`GameScreen.tsx`)
- **amandava プレイ時**:
    - 選択されたランダムな対象に「水（WATER）」または「墨（SUMI）」のエフェクトを表示。
- **amandava 進化時**:
    - 相手フォロワー全員にダメージエフェクトを表示。

## 4. カード能力調整・修正計画
### 4.1. カードパラメータ変更 (`engine.ts`)
- **あじゃトークン版なゆた (`c_nayuta_ward`)**:
    - スタッツを `4/4` から `3/3` に変更。
- **つぶまる (`c_tsubumaru`)**:
    - 説明文に「進化時：「退職代行」を手札に加える」を追加。
    - `EVOLVE` トリガーを追加し、`GENERATE_CARD` で `s_resignation_proxy` を生成。
    - デッキに含まれるように初期デッキ定義を確認（現在は `MOCK_CARDS` の定義順やデッキ生成ロジックに依存している可能性があるため要チェック）。
- **ルイ・ユー (`c_ruiyu`)**:
    - 効果を全面変更。
    - ファンファーレ: `HEAL_LEADER` (value: 4)
    - 進化時: `HEAL_LEADER` (value: 4)

### 4.2. 進化アニメーション改善 (`GameScreen.tsx`)
- **現状**: 進化演出用の拡大カードが表示されている間も、元のカードが盤面に表示され続けている。
- **修正**: `evolveAnimation` ステートが存在し、かつ `evolveAnimation.card.instanceId` と一致する盤面のカードに対して、一時的に `opacity: 0` または `visibility: 'hidden'` を適用するスタイル処理を追加する。
