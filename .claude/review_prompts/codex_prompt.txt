## 問題1: 「せんか」のオーラ効果（常在効果）の実装

現在のカード定義:
```typescript
{
    id: 'c_senka_knuckler', name: 'せんか', cost: 8, type: 'FOLLOWER',
    triggers: [
        {
            trigger: 'FANFARE',  // ← これが問題。常在効果であるべき
            effects: [
                { type: 'GRANT_PASSIVE', targetPassive: 'STORM', targetType: 'ALL_FOLLOWERS', conditions: { tag: 'Knuckler' } }
            ]
        }
    ]
}
```

問題点:
1. ナックラーへの疾走付与は「ファンファーレ」ではなく「常在効果（オーラ）」であるべき
2. 常在効果は「せんか」が場にいる間、常に味方のナックラーに疾走を付与し続ける
3. 「せんか」が場を離れたら、付与した疾走も消える

現在のエンジンコードには既に「せんかのオーラ効果」を処理するコードが複数箇所にある:
- カードプレイ時（1542行目）
- トークン召喚時（2266行目）

しかし、カード定義自体がFANFAREになっているため、ファンファーレとしても発動してしまう。

提案する修正:
1. カード定義から FANFARE トリガーを削除
2. 代わりに passiveAbilities に 'AURA' を追加
3. オーラ効果の詳細は auraEffect フィールドで定義

## 問題2: エフェクト表示の整合性チェック

相手が出したカードや特殊召喚されたカードに、バリア・オーラ・守護などのエフェクトが表示されない場合がある。
効果自体は発動しているが、ビジュアルエフェクトが欠落している。

考えられる原因:
1. 状態更新のタイミングでエフェクト計算が行われていない
2. 特殊召喚時にパッシブ能力の再計算がされていない

提案する解決策:
1. ボード上のカードのパッシブ能力を毎フレーム（または状態変更時に）再計算
2. オーラ効果の影響を考慮したパッシブ能力の動的計算
