# 05_DigitalCardGame プロジェクトルール

## Gitコミットルール

// turbo-all

1. **定期的なコミット**: 以下のタイミングで必ずGitにコミットする
   - 大規模な機能を実装した後
   - ある程度の機能のまとまりを実装した後
   - バグ修正が完了した後
   - UIの大きな変更を行った後
   - 作業終了時

2. **コミットメッセージ**: 日本語で記述し、変更内容を明確に説明する

3. **コミットコマンド**:
   ```
   cmd /c "git add -A"
   cmd /c "git commit -m コミットメッセージ"
   ```

## 開発ワークフロー

1. 新機能実装前に既存コードを確認
2. 実装計画書（implementation_plan.md）を更新
3. 変更を実装
4. 動作確認
5. **Gitにコミット**
6. walkthrough.mdを更新

## 注意事項

- デグレを防ぐため、既存機能への影響を常に確認
- 要件は必ず実装計画書にメモ
- 大きな変更の前は必ずコミットしてバックアップ
